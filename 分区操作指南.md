## 将 F 盘空间合并到 C 盘 — 操作指南

### 当前状态

F: 分区已通过 diskpart 成功删除，磁盘上释放了约 **173GB 未分配空间**。
但由于 C: 和原 F: 之间隔着 D: 和 E:，需要移动分区才能让 C: 使用这块空间。

傲梅分区助手 (AOMEI Partition Assistant) 已经安装并打开。

### 当前磁盘布局（Disk 0）

```
[EFI系统] [MSR保留] [C: Windows 195GB] [D: Data 408GB] [E: PerFiles 176GB] [未分配 173GB] [恢复 1GB]
```

### 目标布局

```
[EFI系统] [MSR保留] [C: Windows ~368GB] [D: Data 408GB] [E: PerFiles 176GB] [恢复 1GB]
```

---

### 操作步骤（共 4 步）

#### 第 1 步：将 E: 分区向右移动

1. 在傲梅分区助手主界面的分区列表中，**右键点击 E: (PerFiles) 这一行**
2. 在弹出菜单中选择 **"调整/移动分区"** (Resize/Move Partition)
3. 在弹出的窗口中，你会看到一个分区条和两侧的可拖拽区域
4. **把整个 E: 分区块拖到最右边**（让它移到原来 F: 的位置）
5. 点击 **"确定"**

> 此时未分配空间应该出现在 D: 和 E: 之间

#### 第 2 步：将 D: 分区向右移动

1. **右键点击 D: (Data) 这一行**
2. 选择 **"调整/移动分区"** (Resize/Move Partition)
3. **把整个 D: 分区块拖到最右边**
4. 点击 **"确定"**

> 此时约 173GB 未分配空间应该紧挨在 C: 的右侧

#### 第 3 步：扩展 C: 分区

1. **右键点击 C: (Windows) 这一行**
2. 选择 **"调整/移动分区"** (Resize/Move Partition)
3. **把 C: 分区的右边界拖到最右边**，将所有未分配空间并入 C:
4. 点击 **"确定"**

> C: 分区应该从 195GB 变为约 368GB

#### 第 4 步：应用所有操作

1. 点击界面左上角的 **"应用"** (Apply) 按钮（蓝色带勾号的按钮）
2. 在弹出的确认窗口中，检查操作列表是否正确
3. 点击 **"执行"** (Proceed)
4. 如果提示需要重启电脑进入 PreOS 模式，点击 **确认重启**

---

### 重要提示

- 整个过程大约需要 **10-30 分钟**
- 操作期间 **请保持电源连接，绝对不要中断**
- 如果某一步提示需要付费版 (Pro)，请截图告诉我，我帮你换其他工具
- 重启后电脑会自动在 PreOS 模式下完成分区移动，完成后会自动进入 Windows

### 如果遇到问题

如果 AOMEI 免费版不支持"移动分区"功能，可以使用以下替代工具：
- **DiskGenius** 免费版：www.diskgenius.com
- **MiniTool Partition Wizard** 免费版：www.partitionwizard.com

操作方式基本相同：右键分区 → 移动 → 拖到右边 → 应用。
